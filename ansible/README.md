# Ansible Automation for K8s The Hard Way

Эта директория содержит автоматизацию для развертывания компонентов Kubernetes на виртуальные машины, созданные через Terraform.

## Структура проекта

* `master.yml` — главный плейбук. Запускает весь процесс установки по порядку.
* `playbooks/` — логические этапы установки (подготовка ОС, ETCD, Control Plane, Workers).
* `templates/` — Jinja2 шаблоны для системных юнитов и конфигов Kubernetes.
* `inventory.ini` — динамически обновляемый файл хостов (управляется Terraform).

## Порядок развертывания

Перед запуском убедитесь, что:

1. Виртуальные машины запущены (`terraform apply`).
2. Сертификаты и kubeconfigs сгенерированы в родительской директории `certs/`.

### Запуск всего цикла:

```bash
ansible-playbook master.yml
```

### Запуск отдельных этапов:
Если нужно пересобрать только воркеры:

```bash
ansible-playbook playbooks/workers.yml
```

## Важные настройки (ansible.cfg)

* `host_key_checking = false`: Отключено для удобства работы с локальными ВМ, которые могут часто пересоздаваться.
* `interpreter_python = auto_silent`: Автоматический выбор Python на целевых узлах Ubuntu 24.04.

## Особенности этапов

1. **Prepare**: Отключает SWAP (критично для kubelet), настраивает параметры ядра (br_netfilter, ip_forward) и устанавливает базовые зависимости.
2. **Distribute**: Копирует сертификаты и конфиги. Пути в плейбуке настроены на локальную директорию `../certs/`.
3. **ETCD**: Разворачивает кластер из 3-х узлов. Использует статические IP, определенные в инвентаре.
4. **Control Plane**: Поднимает API-Server, Scheduler и Controller Manager.
5. **Workers**: Устанавливает containerd, runc и настраивает kubelet. Для каждого воркера автоматически вычисляется его Pod CIDR на основе индекса в имени (например, worker-0 -> 10.200.0.0/24).
