# Terraform State Backend (Minio)

Данный компонент обеспечивает работу S3-совместимого хранилища для хранения state-файлов Terraform. Это позволяет избежать конфликтов при совместной работе и сохраняет историю инфраструктуры вне локальной директории `.terraform`.

## Конфигурация

Сервис использует переменные окружения, определенные в корневом файле `.env.local`. Перед запуском убедитесь, что в нем экспортированы значения:

```bash
# Секреты для контейнера Minio
export MINIO_ROOT_USER="admin"
export MINIO_ROOT_PASSWORD="password"

# Секреты для взаимодействия Terraform с Minio
export AWS_ACCESS_KEY_ID="$MINIO_ROOT_USER"
export AWS_SECRET_ACCESS_KEY="$MINIO_ROOT_PASSWORD"
```

## Быстрый старт
 1. **Загрузка окружения и запуск**:
   Перейдите в директорию компонента и запустите контейнер:

```bash
source ../.env.local
docker compose up -d
```
  
 2. **Подготовка бакета**:

   После первого запуска необходимо создать бакет, в который Terraform будет складывать стейт.
   * Откройте консоль: http://localhost:9001
   * Логин/Пароль: из вашего .env.local
   * Создайте бакет с именем: terraform-state (или тем, что указано в providers.tf).

## Особенности реализации

 * **Persistent Data**: Все данные Minio хранятся в локальной директории ./minio_data. Она добавлена в .gitignore.
 * **Healthcheck**: В docker-compose.yml настроена проверка состояния. Контейнер не будет считаться готовым, пока API (порт 9000) не начнет отвечать.
 * **Security**: Доступ к консоли управления выведен на порт 9001. Рекомендуется не изменять MINIO_PROMETHEUS_AUTH_TYPE: "public" только для локальных тестов.
 
## Обслуживание и очистка
Если вам нужно полностью сбросить состояние хранилища (вместе со всеми стейтами инфраструктуры):
 1. Остановите контейнер:
   
```bash
docker compose down
```
  
 2. Удалите директорию с данными:
   
```bash
rm -rf ./minio_data
```

*Внимание: это действие необратимо уничтожит terraform.tfstate!*
